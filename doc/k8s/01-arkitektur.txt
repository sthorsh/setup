Deklarativ modell
  Desired state: ønsket tilstand for clusteret 
  Manifest: beskriver desired state (yaml, json)
  Current state: clusterets faktiske tilstand
  Drift: current tilstand != desired state
  Rectify: k8s forsøker å gjenopprette desired state

Fordeler ved deklarativ modell
  Selvdokumenterende
  Spec-once deploy-many
  Versjonskontroll (CVS)

  Noder

K8s objekter
  Proper 1st class rest objects in k8s api
  Definert i form av YAML el JSON manifest
  Deployes ved å sende manifest til apiserver
