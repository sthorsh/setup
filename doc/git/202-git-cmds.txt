Refspec
  Refspec: [+]SRC:DST
  SRC: which refs to fetch
  DST: which local refs to update
  Default: remote.origin.fetch feks +refs/heads/*:refs/remotes/origin/*

    - Objekter dvs objects/ (objektdb)

Pull
  From git fetch --help line 351:
  There is a difference between listing multiple <refspec> directly on git pull command line and having multiple
  remote.<repository>.fetch entries in your configuration for a <repository> and running a git pull command without any
  explicit <refspec> parameters.  <refspec>s listed explicitly on the command line are always merged into the current
  branch after fetching. In other words, if you list more than one remote ref, git pull will create an Octopus merge. On
  the other hand, if you do not list any explicit <refspec> parameter on the command line, git pull will fetch all the
  <refspec>s it finds in the remote.<repository>.fetch configuration and merge only the first <refspec> found into the
  current branch.  This is because making an Octopus from remote refs is rarely done, while keeping track of multiple
  remote heads in one-go by fetching more than one is often useful.
