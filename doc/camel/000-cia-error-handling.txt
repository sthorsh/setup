Klassifisering
  Recoverable feil
  Irrecoverable feil

Recoverable feil
  exchange.getException(): Exception
  exchange.setException(Throwable): void
  Clib: ClibRecoverableException

Irrecoverable feil
  exchange.getException(): Exception
  exchange.setException(Throwable): void
  Clib: ClibIrrecoverableException

Irrecoverable errors
  exchange.getIn|Out().getFault(): boolean
  exchange.getIn|Out().setFault(true): void
  Clib: SOAP?

Tommelfingerregler
  Camel anser alle feil som recoverable (camel error handling pattern)
  Camel trigger error handler kun hvis exchange.getException() != null

Camel error handlers
  DefaultErrorHandler 
  NoErrorHandler 
  DeadLetterChannel

DefaultErrorHandler extends RedeliveryErrorHandler 
  Gir opp
  Stopper ruting av meldingen
  Propagerer exchange/feil «bakover»
  Logger routetrace og stacktrace
  Evt forsøk på nytt: redelivery policy
  Evt skop
  Evt exception policy
  Evt error handling

NoErrorHandler
  Et tomt skall uten logikk

DeadLetterChannel extends RedeliveryErrorHandler 
  Undertrykker feilen
  Camel stopper ruting av meldingen
  Flytter feilet|opprinnelig melding til en DLQ (log, fil, database)
  Den som kaller anser meldingen for prosessert
  Evt forsøk på nytt: redelivery policy
  Evt skop
  Evt exception policy
  Evt error handling

----------

Håndtering
  Propagere feil bakover (gi opp, som Java exception)
  Fjerne exception fra exchange, lagre den som property (undertrykke feilen)
  Prøve igjen (redelivery/retry)
  Rute til en annen rute (detour vha exception policies)
  ...
