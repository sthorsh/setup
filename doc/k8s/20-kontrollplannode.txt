API
  API-objekter: representerer tilstand
  Imperativ konfigurasjon: vha kommandoer
  Deklarativ konfigurasjon: vha manifester
  Implementasjon: HTTP verb dvs CRUD, RESTfullt API, JSON

API-server
  Clusterets aksessupunkt
  Mottar eksterne og interne requests
  Alle konfigurasjonsendringer går gjennom API-serveren
  Sender nye og oppdaterte API-objekter til etcd for persistens

Controller manager
  Lytter på api-server for nye pods
  Manager for controllere aka controller loops
  Implementerer lifecycle functions for controllere
  Controllere definerer clusterets desired state feks Deployment
  Controllere overvåker clusterets running state
  Controllere overvåker feks Pod state: er pod up and running?
  Controllere overvåker feks Pod health vha prober: er app i pod up, running og healthy?
  Controllere oppdaterer API-server med ny konfigurasjonen når desired state != running state

Scheduler
  Lytter på api-server for ikke-schedulerte pod'er
  Hensyntar administrative policies
  Hensyntar affinitet og anti-affinitet
  Hensyntar ressurser som etterspørres i manifester
  Hensyntar tilgjengelige ressurser på nodene
  Schedulerer dvs tilordner nye pod'er til noder
  
etcd
  Clusterets datalager
  Lagrer API-objekter som key-value par

CoreDNS
  En add-on pod
  DNS serveren som brukes inne i clusteret
  Hver pod er konfigurert med (1) ip-addr til fe-svc for coredns og (2) search suffix domain
  Pods, svcs og noder registrerer sin ip-addr i coredns når de opprettes
  Apps bruker coreDNS for service discovery

Kubelet
  Overvåker /etc/kubernetes/manifests/
  Ruller ut pods iht statiske manifests

Kube-proxy
  En pod
  Lytter på api-server for endringer i nettverkstopologi
  Vanligvis implementert vha iptables
  Implementerer appservices
  Ruter apprequester inn i clusteret til riktig pod
  Implementerer lastbalansering over N pods bak en fe-svc
  
Containerd
  Pulls images fra container image register
  Starter og kjører containere

Calico
  En add-on pod (aka plugin)
  Implementerer et podnettverk

OS
  Swap off
