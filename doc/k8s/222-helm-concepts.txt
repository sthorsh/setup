HELM

Helm
  En pakkemanager
  
Hvorfor
  Helm opererer på chart/release (objekter) som en entitet → enkelt, raskt, trygt
  Helm templating → fleksibelt
  Pakker, deps, repos → gjenbruk
  ...
    
Helm v2
  https://v2.helm.sh/docs
  Klient: helm
  Server: tiller (pod)
  Kommunikasjon: helm → tiller → apiserver
  Tiller kjører med rollen "cluster-admin"
  Krav: en tiller service account med færre rettigheter
  Krav: kryptering med SSL av gRPC mellom helm og tiller
    
Helm v3
  https://v3.helm.sh/docs/
  Klient: helm
  Kommunikasjon: helm → apiserver
  
Helm chart
  Et katalogstruktur
  
Helm chart struktur
  Chart: ett chart, N objekter (standalone)
  Umbrella chart: ett parent chart, M subcharts, N objekter
  
Templates
  En mal for et manifest
  En template engine prosesserer malen
  Direktiver ersattes med data (values) 
  Direktiver ersattes med kode (logikk)
  Template engine output'er et manifest
  Cf https://v3.helm.sh/docs/chart_template_guide/
    
Helm release
  En instans av et helm chart
  E.g ett chart, en release, ett cluster
  E.g ett chart, flere releaser, samme cluster e.g db (konflikt)
  E.g ett chart, flere releaser, ulike cluster e.g dev/test/prod
  
Helm chart package
  Et arkiv
  E.g frontend-1.2.2.tgz

Avhengigheter
  Subcharts
  Eksterne charts
  
Helm repositories
  En http server
  E.g apache, nginx, chartmuseum
