Git er et VCS
  Distribuert VCS
  VCS
  Content tracker
  Persistent map: ~/,git/objects

Git er et filsystem
  Fil: blob
  Katalog: tree
  Versjon: commit

Objekter
  Commit, tree, blob, tag
  Alle objekter er immutable

Commit
  Er et snapshot/freeze-frame av prosjektet
  Har en peker til et tree-objekt (snapshotet)
  Har en peker til parent commit(s) (historikken)
  Unreachable commit: cannot be accessed through a branch or tag

Tag
  Har en peker til en commit
  Er som en branch som aldri flytter seg

Branch
  Er en ref/peker til en commit
  Er et entry punkt til en history of commits

HEAD
  Er en ref/peker til en branch (ref-til-ref)
  Vanligvis: HEAD → current branch → current commit
  Eller er en ref/peker direkte til en commit (detached HEAD)

Merge
  Merge commit: ny commit med to parent commits (source, target)
  Fast-forward merge: move current branch tip to target branch tip

Rebase
  Alternativ til merge
  Gjenta commits on top of another base tip

Distribusjon
  Maskin1: repo (remote: maskin2)
  Maskin2: klone av repo (remote: maskin1)
  Ved kloning: git oppretter remote origin
  Et prosjekt kan ha ≥1 remotes: $ git remote add <new_remote> <url>
  Hver remote kan ha ≥1 urls: $ git remote set-url --add <remote> <new_url>

Kloning og branches
  Origin: master branch
  Lokalt: master branch
  Lokalt: origin/master branch (i synk med remote origin master branch)
