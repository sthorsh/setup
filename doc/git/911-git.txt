GIT CMDs

Spør for å forstå en kommando
  Hvordan flytter kommandoen data?
  Hvordan endrer kommandoen repositoriet?

git
------------------------------------------------------
$ git add <file>                cp data tree → index
  $ git am
  $ git annotate
  $ git apply
  $ git archive
  $ git bisect
  $ git blame <file>              explore history
  $ git branch [[-d] name]        list|create|delete branch
  $ git bundle
  $ git checkout [branch]         mv HEAD in repo; cp data tree ← index ← repo
  $ git cherry
  $ git cherry-pick
  $ git clean
  $ git clone                     cp data tree ← index ← repo ← remote_repo
$ git commit                    cp data index → repo; update repo objects and refs
  $ git config
  $ git describe
$ git diff [--cached]           diff(tree,index) or diff(index,repo)
  $ git difftool
  $ git fetch <remote>            cp data repo ← remote_repo
  $ git filter-branch
  $ git format-patch
  $ git fsck
  $ git gc
  $ git get-tar-commit-id
  $ git grep
  $ git help
  $ git imap-send
$ git init                      init repo
  $ git instaweb
  $ git interpret-trailers
  $ git log                       explore history
  $ git merge <source> <target>   merge from source branch to target branch
  $ git mergetool
  $ git mv <file₁> <file₂>        mv file in tree; update index with new name
  $ git name-rev
  $ git notes
  $ git pull <remote>             fetch; merge
  $ git push <remote> <branch>    cp data repo → remote_repo
  $ git rebase [-i]               swiss army knife
  $ git reflog HEAD               explore history
  $ git remote [-v]
  $ git repack
  $ git replace
  $ git request-pull
  $ git reset                     mv current branch; hard|mixed|soft reset
  $ git revert HEAD
  $ git rm <file>                 rm file from tree and index
  $ git shortlog
  $ git show                      explore history
  $ git show-branch
  $ git stage
  $ git stash                     cp data stash ← tre & index; checkout current commit
$ git status                    clean, untracked, unstaged|staged changes
  $ git submodule
  $ git subtree
  $ git tag
  $ git verify-commit
  $ git whatchanged
  $ git workspacetree

The golden rule of git
  Aldri endre pushed history      commit --amend, rebase -i, revert
